## Scope Delivered

- Notifications API for authenticated users
- Dashboard notifications page to review and manage updates
- Mark single notification as read
- Mark all unread notifications as read
- Navigation integration in dashboard

## API Endpoints

- `GET /notifications`
  - Returns latest notifications for authenticated user (up to 100)
- `POST /notifications/:id/read`
  - Marks one notification as read
- `POST /notifications/read-all`
  - Marks all unread notifications as read for the current user

## Frontend

- New page: `/dashboard/notifications`
- Shows unread count, notification list, and quick actions
- Supports:
  - Mark individual notification as read
  - Mark all notifications as read

## Notes

- Notifications are already generated by hearing scheduling/status updates from Phase 7.
- This phase provides the user-facing inbox and APIs to consume/manage those events.
